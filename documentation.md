# Документация проекта PlayPick

## 1. Обзор
CPlay — десктопное (кроссплатформенное) приложение для подбора видеоигр, разработанное на Python с использованием KivyMD. Позволяет пользователям фильтровать игры по жанрам и платформам, а администраторам — управлять списком игр.

## 2. Технологический стек
- Python 3.11
- Kivy 2.3.1
- KivyMD 2.0.1 (Material Design виджеты)
- SQLite3 (встроенная БД)

## 3. Функциональные возможности
### 3.1. Пользовательские роли
- **Гость** – просмотр и фильтрация игр без регистрации.
- **Пользователь** – полный доступ к фильтрации после входа.
- **Администратор** – дополнительный доступ к панели управления играми (просмотр, добавление/удаление – в разработке).

### 3.2. Основные экраны
- **LoginScreen** – форма входа (поля: имя пользователя, пароль; кнопки: «Войти», «Регистрация», «Войти как гость»).
- **RegisterScreen** – форма регистрации (поля: имя пользователя, пароль, подтверждение пароля; кнопки: «Зарегистрироваться», «Назад к входу»).
- **GamesScreen** – главный экран со списком игр и чипами фильтрации.
- **AdminScreen** – панель администратора (список всех игр, кнопки управления).

### 3.3. Фильтрация игр
- Используются **чипы типа filter** (`MDChip` с `type="filter"`).
- Чипы разделены на две горизонтальные прокручиваемые панели: «Жанры» и «Платформы».
- При выборе любого чипа автоматически обновляется список игр (активные фильтры комбинируются по логике **ИЛИ** внутри категории и **И** между категориями).
- Состояние чипов отслеживается через свойство `active`. Обработчик `on_chip_active` вызывает `apply_filters()`, который собирает активные значения и передаёт их в `fetch_games_by_criteria()`.

## 5. База данных (SQLite)
### 5.1. Схема таблиц
- **roles** – `id`, `name` (user, admin).
- **users** – `id`, `username` (unique), `password`, `role_id` (FK), `created_at`.
- **genres** – `id`, `name` (unique).
- **platforms** – `id`, `name` (unique).
- **games** – `id`, `title`, `description`, `rating`, `created_at`.
- **game_genres** – связь many-to-many: `game_id` (FK), `genre_id` (FK), составной primary key.
- **game_platforms** – аналогично для платформ.

### 5.2. Ключевые функции (models/database.py)
- `init_db()` – создаёт таблицы и заполняет начальными данными (жанры, платформы, тестовые игры).
- `register_user(username, password, role='user')` – создаёт нового пользователя.
- `login_user(username, password)` – проверяет учётные данные, возвращает словарь пользователя.
- `fetch_games_by_criteria(genres=None, platforms=None)` – возвращает список игр, соответствующих фильтрам. Параметры — списки названий жанров/платформ. Внутри использует JOIN и подзапросы.
- `get_all_genres()`, `get_all_platforms()` – для построения чипов и админки.
- `add_game()`, `delete_game()` – для административных функций.

## 6. Интерфейс пользователя (ключевые элементы)
### 6.1. Экран Games (games.kv)
- Верхняя панель `MDTopAppBar` с заголовком и кнопками:
  - **Выход** (logout) – для всех.
  - **Шестерёнка** (admin) – только для администратора (проверка через `app.is_admin()`).
- Горизонтальные прокручиваемые панели чипов:
  - `MDScrollView` с `do_scroll_x=True`, внутри `MDBoxLayout` с `id: genres_chips` / `platforms_chips`.
- Список игр: `MDScrollView` + `MDList` с `id: games_list`. Каждый элемент — `OneLineListItem`.

### 6.2. Экран Admin (admin.kv)
- Простой список всех игр (из `fetch_games_by_criteria()`).
- Верхняя панель с кнопкой «Назад» и «Добавить» (заглушка).

## 7. Логика фильтрации (games_screen.py)
- **create_chips()** – создаёт чипы, присваивает каждому атрибуты `filter_type` и `filter_value`, привязывает `on_chip_active`.
- **on_chip_active(chip, active)** – вызывается при изменении состояния чипа, запускает `apply_filters()`.
- **apply_filters()** – проходит по всем чипам, собирает активные значения, вызывает `load_games(selected_genres, selected_platforms)`.
- **load_games(genres, platforms)** – выполняет `fetch_games_by_criteria` и обновляет `games_list`.

## 8. Установка и запуск
1. Клонировать репозиторий.
2. Создать виртуальное окружение (рекомендуется).
3. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустить приложение:
   ```bash
   python main.py
   ```
5. При первом запуске автоматически создаётся файл `data/games.db` с тестовыми данными.

## 9. Тестовые учётные записи
По умолчанию после инициализации БД пользователей нет. Чтобы создать администратора, можно выполнить вручную:
```python
from models.database import register_user
register_user('admin', 'adminpass', role='admin')
```
Либо через интерфейс регистрации (все новые пользователи получают роль 'user').

## 11. Примечания
- Проект использует **KivyMD 2.0.1**, поэтому импорты виджетов соответствуют этой версии (например, `MDChip` из `kivymd.uix.chip`).
- Для работы с чипами критично указывать `type="filter"` и использовать свойство `active`.
- Горизонтальная прокрутка чипов реализована через `MDScrollView` с фиксированной высотой.

Документация актуальна на момент версии 1.0.0.
